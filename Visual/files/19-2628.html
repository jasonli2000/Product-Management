PRCS REC.DATE ALL (2628)    OPTION (19)

Name Value
NAME PRCS REC.DATE ALL
MENU TEXT All Transactions with Final Partials
UPPERCASE MENU TEXT ALL TRANSACTIONS WITH FINAL PA
DESCRIPTION 
Program loops through all transactions for a selected control point.
For each transaction read, program checks whether P.O. has been entered,
and the date received has not yet been entered.  It then checks the
partials.  If the final partial has been received, or the P.O. is
either a Certified Invoice, Direct Delivery or Payment in Advance,
the user is allowed to enter the Date Received by the Service.
CREATOR USER,ONE
TYPE run routine
ROUTINE EN1^PRCSRCD
