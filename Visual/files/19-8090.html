LR ORDERED TESTS BY PHY (8090)    OPTION (19)

Name Value
NAME LR ORDERED TESTS BY PHY
MENU TEXT ORDERED TEST COST BY PROVIDER
UPPERCASE MENU TEXT ORDERED TEST COST BY PROVIDER
DESCRIPTION 
This option provides a listing of tests sorted by combination of ordering
providers and locations. The user can select any date range.
The option will list for each sort criteria the test, test price/cost,
and urgency subtotals and totals. A detailed listing is also available.
 
NOTE: This option uses the data stored in LAB ORDER ENTRY (#69) to produce
the report. If Orders and Accessions have been purged for the date range
selected, a report will not be generated. Selected date range should be
within the limits of stored data.
 
This report can be generated using either test price or cost. The value
for price or cost is determined locally and the appropriate file is
populated with a dollar figure.
 
The report's price or cost is determined by using series of fields to
identify the price/cost. The first field is the SP COST field (#1) or SP
PRICE (#2) of the SPECIMEN subfile multiple (#64.01) of the WKLD CODE file
(#64). If no price or cost is found, the next fields checked are the COST
field (#7) or COST (#8) of the File #64. If no price/cost is found in
these fields, the last field checked is the TEST COST field (#1) of the
LABORATORY TEST file (#60). Default value for PRICE/COST is $1.00.
 
Depending on the sort scheme, locations can be either Hospital Location or
Institutions. The user is also allowed to enter FREE TEXT non-standard
locations. If the
report should contain all locations or providers, bypass the prompt for
specific data.
   
There are several methods available to sort the data for the report.
   
This report maybe useful to track tests done by hosptial locations,
institutions or providers.
