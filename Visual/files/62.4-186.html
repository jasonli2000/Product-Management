Re Setting files for operation:
  AUTO INSTRUMENT FILE:
     When entering tests into the CHEM TESTS field be sure the PARAM 3
     subfields have no spaces in the test names entered here.
  LOAD/WORKLIST FILE:
     In order to display the tests for verification using EA in the
     same order as they come off on the EPX printouts you must:
     1)  Create a test in the LAB TEST FILE (e.g. EPX PROFILE) with
         the tests in LAB TESTS INCLUDED IN PANEL entered in the order
         they display from your EPX printout.  TYPE is OUTPUT.
     2)  Enter this test into the LOAD/WORKLIST Profile you have
         given for the EPX.  This test should be BUILD NAME ONLY: NO.
         All the other tests should be set to BUILD NAME ONLY: YES.
     3)  LOAD TRANSFORM should be set to UNIVERSAL.  INITIAL SETUP
         needs to be set to INITIAL SET.
From ABBOTT
     In regard to smooth operation of the Downloading routine:
     1)  Must reindex the disk at least weekly, if not more frequently.
     2)  Use a separate carousel (e.g. 99) for running controls,
         calibrators, and water.  If system prompts for these
         parameters should appear during a run on your test
         carousel they will interfere with the download.
     3)  You may use the same carousel until it is full.  The routines
         will progress to the next carousel.  Do not use a carousel
         that has previously been employed without first doing the
         reindex (see #1 above).
     4)  Auto send- ON
     5)  The rationale for the reindexing is that current EPX software
         will not consistently delete the PID which is assigned during
         the download.  This causes problems when trying subsequent
         downloads to the same carousel.  Hence, reindex or use
         a different carousel.
     6)  Everytime there is a glitch with the host computer the
         EPX stops talking to the host and must be rebooted.
