DSIC DM ADD/DELETE QUEUE (1595)    REMOTE PROCEDURE (8994)

Name Value
NAME DSIC DM ADD/DELETE QUEUE
TAG UPD
ROUTINE DSICDM
RETURN VALUE TYPE SINGLE VALUE
AVAILABILITY AGREEMENT
DESCRIPTION 
This will allow you to add or delete an entry to file 19621.  When you 
use the VistA Imaging import an image API, you are returned a queue 
number.  This number means that VistA Imaging background processor 
accepted your request and place it a queue and will process it in due 
time.  After you get this successful queue message you should immediately 
file that data through this RPC.
 
This file is to allow communication between the VistA Imaging background 
processor and the DSS Document Management system.
INPUT PARAMETER
  1. INPUT PARAMETER:   ACT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   NO
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    This is the action that should be taken.  The default value if not passed 
is A.
  ACT = A then add a record to file 19621 - TRANID and QUEUE required
  ACT = D then delete a record in file 19621 - TRANID is only required
  2. INPUT PARAMETER:   TRANID
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   50
    REQUIRED:   NO
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    This is the transaction ID you passed to the VistA Imaging import API.  
It must be in the format DSS;xxxxxx
 
This is optional.  If it is not passed, then a transaction ID will be 
built from the APP parameter.
  3. INPUT PARAMETER:   QUEUE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   20
    REQUIRED:   NO
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    This is only required if you have successfully queued an import request 
to VistA Imaging.  VistA Imaging import API will return you a QUEUE 
number.  This is only required if you are adding a record to file 19606.
  4. INPUT PARAMETER:   APP
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   NO
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
    This is optional only for DocManager.  For all othr DSS applications, 
this is required.

RETURN PARAMETER DESCRIPTION 
Returns a string:
  If successfully added return 1^message^TRANID
  If successfully deleted return 1^message
  If unsuccessful return -1^error message
