DSIC ACTIVE PERSON CLASS (1697)    REMOTE PROCEDURE (8994)

Name Value
NAME DSIC ACTIVE PERSON CLASS
TAG PER
ROUTINE DSICDUZ
RETURN VALUE TYPE SINGLE VALUE
AVAILABILITY AGREEMENT
DESCRIPTION 
This will return a user's active person class for a given date.
INPUT PARAMETER
  1. INPUT PARAMETER:   USER
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   15
    REQUIRED:   NO
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    This is the user's DUZ value whose active person class is desired.
  2. INPUT PARAMETER:   DATE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   14
    REQUIRED:   NO
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    This is a Fileman date.time for the date to check for an active person 
class membership.
RETURN PARAMETER DESCRIPTION 
Return -1^message if problems or no active person class
Else return p1^p2^p3^...^p8  where
     p1 = IEN to file 8932.1     p5 = Effective date
     p2 = Occupation             p6 = expiration date
     p3 = specialty              p7 = VA Code
     p4 = sub-specialty          p8 = specialty code
