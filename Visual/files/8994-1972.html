OOPS SIGNATURE VALIDATION (1972)    REMOTE PROCEDURE (8994)

Name Value
NAME OOPS SIGNATURE VALIDATION
TAG DECODE
ROUTINE OOPSGUIS
RETURN VALUE TYPE SINGLE VALUE
DESCRIPTION 
This broker call will return the SIGNATURE BLOCK PRINTED NAME from the New
Person File (#200) if the validation logic for signing the case passes.
INPUT PARAMETER
  1. INPUT PARAMETER:   IEN
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   10
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    This parameter contains the internal record number for the case.
  2. INPUT PARAMETER:   CALL
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   YES
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    This parameter contains the letter that cooresponds to the menu that was
used to access the option passing in the data.
  3. INPUT PARAMETER:   FORM
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   10
    REQUIRED:   YES
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    This paramter contains the type of form that is being accessed.  As of ASISTS 
version 2, patch 5, the only value expected is CA7.
RETURN PARAMETER DESCRIPTION 
The return value will contain either the SIGNATURE BLOCK PRINTED NAME or an
error message indicating that the electronic signature decoding failed.
