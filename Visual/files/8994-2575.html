NAME DSIF AUTHLIST
TAG AUTHLIST
ROUTINE DSIFENA2
RETURN VALUE TYPE GLOBAL ARRAY
AVAILABILITY AGREEMENT
WORD WRAP ON TRUE
VERSION 3.2
DESCRIPTION 
Pass in IEN (DFN), return list of current authorizations
 
or -1^Not a valid patient
 
or -1^Not a valid patient
INPUT PARAMETER
  1. INPUT PARAMETER:   DFN
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   30
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    Pass in Patient File IEN (file #2)
  2. INPUT PARAMETER:   FLAG
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   NO
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
        FLAG="" OR "0" Outpatient authorizations only (excludes Unauthorized)
    FLAG="1" Inpatient auths onlY
    FLAG="2" Both Outpt, Inpt. & Unauthorized
  3. INPUT PARAMETER:   DSIFDAT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   NO
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
      Enter the to date (in FM format) to restrict the number of records
  returned.
  
  Example: If you enter 3100501 the RPC will return all authorizations up
  to May 1, 2010.
  
  This field can also be used in conjunction with DSIFFDAT to produce a
  specific time period in which authorizations to be returned.
  4. INPUT PARAMETER:   DSIFFDAT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   NO
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
     Enter a from date (in FM format) to limit the authorizations returned.
  
  Example: If you enter 3090101 it will only return authorizations between
  today and January 1, 2009.
  5. INPUT PARAMETER:   DSIF AUTHLIST
RETURN PARAMETER DESCRIPTION
