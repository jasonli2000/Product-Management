DSIF INP PATIENT REG (2624)    REMOTE PROCEDURE (8994)

Name Value
NAME DSIF INP PATIENT REG
TAG REG
ROUTINE DSIFNOT3
RETURN VALUE TYPE SINGLE VALUE
DESCRIPTION 
This RPC will add entries to file 162.2 FEE NOTIFICATION/REQUEST FILE.  
The logic was taken from routine: FBCHREQ* and edit template FBCH ENTER 
REQUEST.
INPUT PARAMETER
  1. INPUT PARAMETER:   DFN
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    Patient File (#2) pointer (Required)
  2. INPUT PARAMETER:   DATE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   14
    REQUIRED:   YES
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    Date and Time (Required - FM format)
Date and Time that the vendor notifies the COJ of this episode of 
hospitalization.
  3. INPUT PARAMETER:   VNDR
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   YES
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    Vendor (Required - Pointer to file 161.2 FEE BASIS VENDOR)
  4. INPUT PARAMETER:   DTOA
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   14
    REQUIRED:   YES
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
    Date & Time of Admission (Required - FM format)
This field contains the date and time that you begin accepting 
responsibility for bills incurred if the veteran is found to be legally 
and medically entitled.
  5. INPUT PARAMETER:   PERS
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   30
    REQUIRED:   NO
    SEQUENCE NUMBER:   5
    DESCRIPTION:   
    This is a free text field that contains the name of the person who is 
notifying you.
  6. INPUT PARAMETER:   IEN
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   NO
    SEQUENCE NUMBER:   6
    DESCRIPTION:   
    Internal Entry Number to file 162.2 FEE NOTIFICATION/REQUEST (Optional - 
Null = Add, Value = Edit). This field is only used for editting an 
existing record.
  7. INPUT PARAMETER:   ADMDX
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   45
    REQUIRED:   NO
    SEQUENCE NUMBER:   7
    DESCRIPTION:   
    This is the admitting diagnosis of the veteran which is usually obtained 
from the person who called.
  8. INPUT PARAMETER:   ATPHY
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   30
    REQUIRED:   NO
    SEQUENCE NUMBER:   8
    DESCRIPTION:   
    Attending Physician (Optional - Free Text)
  9. INPUT PARAMETER:   ATPHPH
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   20
    REQUIRED:   NO
    SEQUENCE NUMBER:   9
    DESCRIPTION:   
    This field contains the phone number of the attending physician while the 
patient was residing in the non-VA facility.
RETURN PARAMETER DESCRIPTION 
Return Values
        -1 ^ Invalid Input!
        -1 ^ This notification has a status of complete.  Cannot edit.
        -1 ^ This Authorization From Date exceeds the 72 hour notification
period.
        NNN = Internal number of newly created record
