DSIF PAT/VEN PAYMENT REPORT (2651)    REMOTE PROCEDURE (8994)

Name Value
NAME DSIF PAT/VEN PAYMENT REPORT
TAG EN
ROUTINE DSIFRPT3
RETURN VALUE TYPE GLOBAL ARRAY
WORD WRAP ON TRUE
DESCRIPTION 
This is an RPC version of the FBPAY routines.
INPUT PARAMETER
  1. INPUT PARAMETER:   FBSORT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    Sort Indicator (1 = Patient, 0 = Vendor - Default = 0)
  2. INPUT PARAMETER:   FBIEN
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   YES
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    Patient/Vendor Internal Entry Number (File 161/161.2 respectively)
  3. INPUT PARAMETER:   FBBDATE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   YES
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    Begining Date for report (FileMan Format)
  4. INPUT PARAMETER:   FBEDATE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   YES
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
    Ending Date for report (FileMan Format)
  5. INPUT PARAMETER:   FB1725R
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   NO
    SEQUENCE NUMBER:   5
    DESCRIPTION:   
    Mill-Bill indicator only used if PROGS passes in Fee Programs with 
internal numbers 2, 3 or 6.
 
M - include only 38 U.S.C. 1725 claims
N - exclude 38 U.S.C. 1725 claims
A - All
  6. INPUT PARAMETER:   PROGS
    PARAMETER TYPE:   LIST
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   YES
    SEQUENCE NUMBER:   6
    DESCRIPTION:   
    List of Fee Programs to include on report (Multi-piece IEN ^ Name)
 
(IEN is pointer to file 161.8 FEE PROGRAMS, the user should only be 
allowed to select active programs.)
RETURN PARAMETER DESCRIPTION 
-1 ^ Invalid Input!
-1 ^ Invalid Input! Mill-Bill!  (This is if program 2, 3 or 6 is passed 
and FB1725 is null)
 
User readable report in return array
