DSIRRPT4 CDD DISCREPANCY RPT (2996)    REMOTE PROCEDURE (8994)

Name Value
NAME DSIRRPT4 CDD DISCREPANCY RPT
TAG CDD
ROUTINE DSIRRPT4
RETURN VALUE TYPE GLOBAL ARRAY
WORD WRAP ON TRUE
DESCRIPTION 
This RPC returns data from the DSIR STATUS HISTORY file to review which
request are being entered as closed on one day yet the status date is a
previous date.
INPUT PARAMETER
  1. INPUT PARAMETER:   FRDT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    FileMan format start date where report will begin looking at stauses. 
Based on status date.
  2. INPUT PARAMETER:   TODT
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   7
    REQUIRED:   NO
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    Last status date to look at for report.
  3. INPUT PARAMETER:   CLRK
    PARAMETER TYPE:   LIST
    MAXIMUM DATA LENGTH:   99
    REQUIRED:   NO
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    Array where each element is equal to the DUZ of selected clerks.
Optional - no entries passed will return all clerks for time period.
  4. INPUT PARAMETER:   SCHED
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   NO
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
    Boolean flag that determines whether a report is scheduled or run
immedieately. 0 - Immediate  1 - Schedule.
  5. INPUT PARAMETER:   ESTART
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   15
    REQUIRED:   NO
    SEQUENCE NUMBER:   5
    DESCRIPTION:   
    Earliest time to start the scheduled task.  Mmust be inFileMan date/time
format.
RETURN PARAMETER DESCRIPTION 
RETURN ARRAY
 if immediate ^ delimited 
 
   Clerk Name ^ Request Name ^ Open Date ^ Closed Date ^ Status Code ^ 
                Status Date ^ Date User Entered Status
 
   or
   if scheduled
   Task^REPORT AFTER yyymmdd.hhmmss
 
   or
   -1^error message
