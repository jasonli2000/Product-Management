MAGV GET PAT ORDERS (3125)    REMOTE PROCEDURE (8994)

Name Value
NAME MAGV GET PAT ORDERS
TAG GETORD
ROUTINE MAGVIM02
RETURN VALUE TYPE ARRAY
AVAILABILITY RESTRICTED
INACTIVE ACTIVE
WORD WRAP ON FALSE
VERSION .9
DESCRIPTION 
Returns an array of CONsult or RADiology orders for an input
ENTERPRISE PATIENT IDENTIFIER.
INPUT PARAMETER
  1. INPUT PARAMETER:   ENTERPRISE PATIENT ID
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   999
    REQUIRED:   YES
    SEQUENCE NUMBER:   1
    DESCRIPTION:   
    ENTERPRISE PATIENT ID.
  2. INPUT PARAMETER:   PATIENT ID TYPE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   YES
    SEQUENCE NUMBER:   2
    DESCRIPTION:   
    Single character per ID TYPE field (#.03) of the
IMAGING PATIENT REFERENCE file (#2005.6).
  3. INPUT PARAMETER:   PATIENT ID ASSIGNING AUTHORITY
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   1
    REQUIRED:   YES
    SEQUENCE NUMBER:   3
    DESCRIPTION:   
    Single character per ASSIGNING AUTHORITY field (#.02) of the
IMAGING PATIENT REFERENCE file (#2005.6).
  4. INPUT PARAMETER:   PATIENT ID CREATING ENTITY
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   30
    REQUIRED:   NO
    SEQUENCE NUMBER:   4
    DESCRIPTION:   
    Free text per CREATING ENTITY field (#.04) of the IMAGING PATIENT REFERENCE file (#2005.6).
  5. INPUT PARAMETER:   ORDER TYPE
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   3
    REQUIRED:   YES
    SEQUENCE NUMBER:   5
    DESCRIPTION:   
    Free text as "CON" for Consult Orders, "RAD" for Radiology Orders.
  6. INPUT PARAMETER:   ORDER DATE FILTER START
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   8
    REQUIRED:   YES
    SEQUENCE NUMBER:   6
    DESCRIPTION:   
    A date formatted as MMDDYYYY.
  7. INPUT PARAMETER:   ORDER DATE FILTER END
    PARAMETER TYPE:   LITERAL
    MAXIMUM DATA LENGTH:   8
    REQUIRED:   YES
    SEQUENCE NUMBER:   7
    DESCRIPTION:   
    A date formatted as MMDDYYYY.
RETURN PARAMETER DESCRIPTION 
On error, returns "-1" followed by an error message. Else returns the number of orders returned, followed by arrayed exam data.
