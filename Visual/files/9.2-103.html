NAME XUFILE ACCESS USE2
HEADER WHO NEEDS FILE ACCESS?
TEXT 
 
VA FileMan users may be granted access to one or more files.  The level of
access authority may range from simply reading data to modifying data
dictionaries.  If a non-programmer user has VA FileMan as a menu option,
that user will not have access to any file until so granted.
 
You, as the systems manager, will need to specify the particular files and
levels of access for your VA FileMan users.  An option has been added to the
User Edit menu for this purpose.  It involves a multiple field in the User
File called 'Accessible File'.
 
Only in special cases will you need to grant file access to users who do not
have VA FileMan privileges.  This might arise if, within the template of an
edit-type option, the user attempts to add or delete entries from a pointed-
to file.  Application programmers may either specify which files you need to
grant to whom, or modify their code or data dictionary specifications.
