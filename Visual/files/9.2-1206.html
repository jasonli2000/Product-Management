                           Dates and Timestamps
 
When entering a NOIS call, the date the problem was reported has to be
entered.  This date is used to create the Call ID.  When closing a call,
the date the problem was resolved is entered.
 
Several date/times are captured when editing a call: the date/time the
call was first entered, the last time editing, the date is captured when
making notes, and when statuses are changed.  All status changes are kept
in kept in a status history file.  Users making status changes are also
stored.  The user making the last edit is also stored.  The date/time
changes are kept in the audit history.
 
Information on timestamp information can be reviewed using the Other
Information action on the Edit screen.
Fields for days since call was entered, last edited, and last status
change can be used in queries.
