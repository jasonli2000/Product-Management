NAME XQLOCKS
HEADER SECURITY KEYS
TEXT 
Security keys can be used to restrict access to options in the menu system.
An option which is locked is not available to users who don't have the key.
Keys are also used within application software packages to control access.
 
Security keys are stored in the Security key file.  They are namespaced by
package.  You can create and provide descriptions for keys with the
[ENTER/EDIT] option.  You can also [ALLOCATE] and deallocate keys with
other options.  The person allocating a key as well as the date allocated
are automatically assigned by the system.
 
A lock appears as a field in the option file.  It can be assigned thru
the [RESTRICT] availability option.
