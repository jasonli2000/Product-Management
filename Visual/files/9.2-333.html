TEXT 
BASIC SEARCH
 
This method checks the entire selected file for duplicates.  It starts
at the beginning of a file and checks each record against a subgroup
of records (defined by the candidate collection routine written by
the package developer) for possible duplicates.  If two records are
determined to be potential duplicates they are placed in the Duplicate
Record file as Potential Duplicates.  These records will then be
verified as either Verified Duplicates or Verified Non-Duplicates.
 
The amount of time the search will take depends on the number of records
in the file.  When you start a Basic search it will be placed in a
Taskman Queue to be run in a background job.  This search can be halted
and restarted any number of times if necessary.
 
 
NEW SEARCH
 
This method checks only newly added and edited records.  It should
be completed in one night, but can be halted and restarted just as
the basic search if necessary.
