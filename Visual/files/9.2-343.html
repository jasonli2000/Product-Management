NAME XDR DUP ALGORITHM
HEADER DUPLICATE RESOLUTION ALGORITHM
TEXT 
The Kernel Duplicate Resolution software uses a algorithm obtained from
the Duplicate Resolution file for determining if two records from the
same file are potential duplicates.
 
In the Duplicate Resolution File entry for a given file there are a
series of tests which will be applied against the two records in
question.  Examples of these tests would be Name, SSN, Mother's Maiden
Name, etc for the VA Patient file.
 
For example in the Name test, the software will take the names from
the two records and compare them and assign them either a
Successful Match Weight or a Unsuccessful Match Weight (defined in
the Duplicate Resolution file) and pass this value back to the
Duplicate Resolution software.
 
Once all of the tests have been applied to the two records in question
a total score will be computed and then checked to see if it exceeds
the Potential Duplicate Threshold% (defined in the Duplicate Resolution
file), if it does the two records will be added to the Duplicate Record
file as Potential, Unverified Duplicates.
