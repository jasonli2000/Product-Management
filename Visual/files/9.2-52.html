Name Value
NAME XQBUILDTREE
HEADER FORCING THE REBUILD OF TREES
TEXT 
This option can be used whenever a modification has been made to items
in a menu requiring that the primary menu trees get rebuilt.  A modification
could be a LOCK added, and option made NOT AVAILABLE, a SYNONYM changed,
etc.  In such cases all primary menu trees need to be rebuilt.
 
This can happen automatically, whenever the ^JUMP feature is used to go
directly to an option, or the rebuild can be 'forced'.  If done auto-
matically, it is done per primary menu tree, per CPU.  Separate copies
of the tree are stored on each CPU.
 
The rebuild process for a single primary menu can take between a few seconds
and several minutes depending on the size of the menu and the state of the
system.
 
The option can be run by [VERIFYING] each primary menu or automatically.
