NAME XQSMD-REPLICATE
HEADER Replicate an exsisting user's delegated options
TEXT 
   You may choose to copy the Delegated Options of one user
to another user.  For instance, if John Smith has been
delegated a set of options to manage you may choose to
delegated that same set of options to another user, say,
Marcie Madona.
   You will also be asked if, after the options set has
been delegated to the second user, Marcie Madona in the example,
those options should be removed from the first user (John
Smith).  If you say no ('N') John Smith will retain the
set of options.  If you say yes ('Y') all Delegated Options
will be removed from John Smith and he will no longer be an
active delegate.
   In order to remove the options from the first user, John
Smith, in this case, you must have a [Delegation Level] lower 
than John Smith.  Your Delegation Level is a number indicating the
steps in the delegation process you are away from the original
delegation of the Site Manager.  The Site Manager will have a
level of 0 (zero).  When she delegates options to someone that
person will have a Delegation Level of 1.  If, in turn, that 
person further delegates options recieved from the Site Manager
to a third person that person will have a level of 2, and so forth.
