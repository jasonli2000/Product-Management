      DESCRIPTION:   
       Some sites are getting errors while trying to install Mental Health patches
 when required mental health builds exist in the REQUIRED BUILDS field (#11) of
 the Kids BUILD file (#9.6).  The install will abort stating that a required
 patch is missing.  However, when the site checks on the missing patch, they
 discover that the patch IS installed.  This became evident when test sites
 were attempting to install YS*5.01*44 which required patch YS*5.01*38 to be
 installed.  The install would abort saying that patch 38 is not installed
 even though it was.  (This same error occurred when test sites were
 attempting to install YS*5.01*45.)
 
 This is caused by old entries in the PACKAGE (#9.4) file confusing the install.
 During installation of a Mental Health patch, the install looks at the
 PREFIX field (#1) of the PACAKAGE file (#9.4).  If it contains "YS", it
 then refers to the history for that package to verify that the required
 patches have been installed.  Any entries that contain the "YS" prefix but
 do not contain "Version 5.01 of Mental Health" in the SHORT DESCRIPTION
 field (#2), cause the install to look at the wrong history.  The install
 aborts because it does not find the required patches.
 
 This patch corrects the problem by executing routine YSPOS46 to "clean up"
 the PACKAGE file.  This routine is included as a post installation routine
 for this patch.  During the installation, the user is asked if they would
 like to 'Delete old package file entries?'  If the user says 'Yes', all but
 the current version 5.01 Mental Health entry is deleted.  If the user
 answers 'No', the old "YS" prefix entries in the "C" cross reference of the
 PACKAGE file (^DIC(9.4,"C","YS",0)) are renamed to YS1, YS2, YS3, etc.  Only
 the current version 5.01 entry is left unchanged.
 
 PLEASE NOTE THAT NOTHING IS INSTALLED WITH THIS PATCH.  IT CONSISTS ONLY OF
 THE USER QUESTION DURING INSTALLATION AND EXECUTION OF POST INSTALLATION
 ROUTINE YSPOS46.

       VHA Directive, 97-059, dated November 25, 1997, "Instituting Global
 Assessment of Function (GAF) Scores in Axis V Mental Health Patients", has
 been released.  This directive provides guidelines for collection of GAF
 Scores for outpatients and inpatients.  Inpatient Axis V (GAF) scores have
 been collected for several years, but up to now there has been no provision
 for collection of GAF Scores for outpatients.  Storage will be in the VISTA
 Mental Health software package.
 
 This patch contains two new API's that are used for the storage/retrieval
 of GAF score data in the DIAGNOSTIC RESULTS - MENTAL HEALTH file (#627.8).
 The API's are described as follows:
 
 API entry point RET^YSGAF(YSX) retrieves GAF score information from the
 DIAGNOSTIC RESULTS - MENTAL HEALTH file (#627.8).  It returns the latest GAF
 score and diagnosis date/time for the patient based on the internal entry
 number (variable YSX) passed into the API.  The score/date is returned in the
 form of a concatenated string.  If no GAF data is found, -1 is returned.
 
 API entry point UPD^YSGAF(YSPN,YSGN,YSGD,YSGC,YSPT) is designed to store
 patient GAF score information in the DIAGNOSTIC RESULTS - MENTAL HEALTH
 file (#627.8).  The following information is passed into this call:
 
        YSPN    Patient name -  Pointer to the PATIENT file (#2)
        YSGN    GAF score (Axis 5)
        YSGD    Date/Time of diagnosis
        YSGC    Diagnosis by -  Pointer to the NEW PERSON file (#200)
        YSPT    Patient type -  'I'npatient or 'O'utpatient
 
 This API updates new field PATIENT TYPE (#66) in the DIAGNOSTIC RESULTS -
 MENTAL HEALTH file (#627.8) which was created to store the patient type of
 the GAF record.  It is a set of codes consisting of 'I' for inpatient and 'O'
 for outpatient.
 
 This API routine call creates a new record with the above information in the
 DIAGNOSTIC RESULTS - MENTAL HEALTH file (#627.8) in order to maintain a
 historical record of GAF scores for an individual patient.

      VHA Directive, 97-059, dated November 25, 1997, "Instituting Global 
Assessment of Function (GAF) Scores in Axis V Mental Health Patients", has 
been released.  This directive provides guidelines for collection of GAF 
Scores for outpatients and inpatients.  Inpatient Axis V (GAF) scores have 
been collected for several years, but up to now there has been no provision 
for collection of GAF Scores for outpatients.  Storage and collection of GAF
scores will be in the VISTA Mental Health software package file
DIAGNOSTIC RESULTS - MENTAL HEALTH (#627.8).  
 
The GAF initiative is using the VISTA HL7 software to transport the GAF 
score data onto the Austin Automation Center (AAC) system.  An A08 HL7 
message will be used to transmit GAF Scores to the National Patient
Care Database (NPCD). This message contains the MSH, EVN, PID and OBX
segments.
 
This patch will allow the sites to transmit GAF scores to the National 
Patient Care Database (NPCD) at the AAC.  Sites will be able to transmit GAF
scores in a batch mode using the call point EN^YSGAFTR.  EN^YSGAFTR will
prompt the user for a date range, this date range will be the selection
criteria that is used when determining the GAF scores to be transmitted to 
the NPCD at the AAC.  The batch processing will allow sites to initially 
populate GAF Scores at the NPCD.
 
After the running of the batch transmission of GAF scores, all GAF scores 
entered will be dynamically sent to the NPCD at the AAC.

The new Q-NPT.DOMAIN.EXT domain, which receives the HL7 messages,
implementation instructions are released by MailMan XM*DBA*119 informational
patch. 
 
The (G)lobal (A)ssessment of (F)unction Transmission patch components are:
        - 4 New routines, YS43POST, YSGAFHL7, YSGAFOBX, YSGAFTR
        - 2 Patched routines, YSDX3B, YSGAF
        - 2 New mail groups, YS GAF TO NPCD, YS GAF TRANSMISSION ACK
        - 2 New Protocols, YS GAF, YS GAF A08 CLIENT
        - 1 New HL7 Application Parameter, YS GAF
        - 1 New HL Lower Level Protocol Parameter, YS GAF SEND
        - 1 New HL Logical Link, YS GAF-SND
 
GAF Mail Groups
---------------
YS GAF TO NPCD: This mail group is used for transmission of HL7 messages 
derived from the parameters defined in the DIAGNOSTIC RESULTS - MENTAL 
HEALTH file (#627.8) to the AAC.
 
YS GAF TRANSMISSION ACK: This mail group is used to receive Confirmation 
and Processing Report Messages from Austin.  
 
Protocols
---------
YS GAF: This event driver protocol defines the associated parameters needed 
to build the HL7 message used to send the GAF data to Austin.
 
YS GAF A08 CLIENT: This subscriber protocol defines the parameter needed by 
HL7 package to determine where to send the HL7 formatted message 
containing the GAF information.

The Mental Health Assistant (MHA) is a graphical user interface for the
most frequently used options within the VistA Mental Health Package,
viz., psychological testing, the Addiction Severity Index (ASI), and
the Global Assessment of Functioning (GAF).
 
All previous tests completed by the selected patient are listed on the
Test Results tab.  A text report is shown for the selected test, and
graphs of numeric scores are available across all instances of a given
test.
 
The Order Tests tab enables the user to specify the staff person
ordering the test and to select from among the tests available to the
person who ordered the test.  Three data entry modes are available:
(1)  Staff entry is optimized for the entry of staff rating data (e.g.,
     the AIMS), which requires the user to see the test questions and
     answers.
(2)  Clerk entry is optimized for transcribing test data from paper
     answer sheets.
(3)  Patient entry is optimized for on-line testing of patients.  The
     patient entry option contains security features to prevent
     unattended patients from using the PC in unauthorized ways.
 
The ASI tab lists all previous ASI interviews and makes it easy to view
either the item report or narrative report for a selected interview.
To help with ASI performance measure compliance, the interval since the
last interview is displayed.  New views of ASI data are the "Domain
Scores" and "Item Trends" buttons, which present graphical and tabular
data across multiple interviews.  The Domain Scores button gives the
user the opportunity to see either problem severity ratings or
evaluation factor scores.  The Item Trends button displays responses to
selected individual items.  It is hoped that these data views will help
with treatment planning and treatment outcome monitoring.  A user-
friendly interface for entering interview data is provided.  This tab
enables staff to quickly enter data, to easily jump from one item to
another, and to enter free text comments at any time.  This tab should
greatly reduce data entry time, whether transcribing interview results
from a paper form or entering them on-line during an interview.
 
The GAF tab affords the user an easy way to enter GAF ratings and to
see previous ratings, which are graphed to indicate trends.  When
entering a new rating, the rating is associated with the GAF rating
criteria.  It is hoped this association will increase the reliability
of ratings and reduce inter-rater variability in the assignment of GAF
ratings.

WHITE BLOOD CELL  (WBC)
 
For reference  "White Blood Cell Count" is a portion of the total blood 
content and the sum total of two different kinds of cells - 
 
         AGRANULOCYTES - those cells which do not have granules
         that absorb a stain.  These cells are subdivided
         into Lymphocytes and Monocytes. 
 
         GRANULOCYTES  -- those cells which contain granules
         that absorb a stain.  These cells are subdivided
         into Neutrophils, Basophils, and Eosinophils.  
 
Both Agranulocytes and Granulocytes are part of the Total White Blood 
Cell Count.  Therefore, the total WBC is the sum of five different types 
of cells.  
 
WBC as well as its five components are usually expressed as the total 
number of cells in a cubic centimeter of blood (cc) or cubic millimeter 
(cu mm or mm-3)or milliliters (mL).  
 
WBC usually falls with a range between 4000 and 11,000 but fluctuates for 
a number of medical reasons.  For brevity, WBC results may be divided by 
1000 and presented as K/cc or K/cu mm or K/mm-3 or K/mL
 
 
The five components are similarly expressed as number of the same type OR 
as a percentage of the total WBC.  
 
The sum of counts of the five components of should equal the total WBC OR 
the sum of the percentages of the five components should equal 100%
 
This test is sometimes also referred to as "Leukocyte Count" 
 
 
Absolute Neutrophil Count (ANC)
 
The Absolute Neutrophil Count is the total number of Neutrophil cells in 
the total WBC count.  The other GRANULOCYTE cells - Basophils and 
Eosinophils (see White Blood Cell Count above) - are not included in this 
number.  Therefore - a test titled GRANULOCYTES - may not produce the 
desired result as it may include Basophils and Eosinophils.
 
Since the ANC is a portion of the WBC, it is only valid if it is 
traceable to the WBC sample drawn at the same date and time.  
 
ANC is a count and not a percentage - expressed as WBC with the total 
number of cells in a cubic centimeter of blood (cc) or cubic millimeter 
(cu mm or mm-3)or milliliters (mL).  Again, this number may be divided by 
a thousand for brevity.  
 
The Absolute Neutrophil Count generally ranges from 1600 to 7700 / but it 
can ever exceed the WBC count.  
 
This test can also be knows as polymorphonuclear cells or PMN's. 
 
 
Neutrophil Percent  (Neut%)
 
The Neutrophil Percent is the portion of the total WBC and that is 
composed of Neutrophil cells, excluding all agranulocytes and other 
granulocytes.  This number is usually between 40% and 70%, but fluctuates 
for many medical reasons.  
 
Since it is reported as a portion of the total WBC, it is only valid if 
it is traceable to the WBC sample drawn at the same date and time.  
 
Neut% is commonly obtained by automatic testing equipment to represent 
the Neutrophil content in the blood.  
 
Neut% is a percentage and not a count - reported in the portion of the 
WBC.  It can be converted into an Absolute Neutrophil Count in the 
following formula (Neutrophil% / 100) X WBC count).  
 
The Neutrophil Percent generally ranges from 37% to 80% and should never 
exceed 90% of the total WBC count unless special medical conditions are 
present.  
 
 
SEGS  & BANDS  
 
The WBC/Granulocyte/Neutrophil cell type can be further subdivided into 
two types.  Mature neutrophils are called SEGS.  Immature neutrophils 
called BANDS also circulate in the blood until they mature into SEGS.  
The sum of SEGS and BANDS is equal to the ANC of a give WBC count.  
 
SEGS and BANDS may be reported as a count, expressed in cc, mm-3, cu mm, 
or mL.  A preceding K indicates the number has been divided by 1000.  
 
When SEGS and BANDS are reported as a number, the sum total of SEGS and 
BANDS equals the total Neutrophil Count (ANC)  NOTE that the program 
needs an identifying test for both SEGS and BANDS.  Reporting SEGS 
without a parallel result from BANDS will result in an error condition 
that may prevent Pharmacy dispense.  
 
This number generally ranges between 1600 and 7700 / cu mm.  It should 
never exceed 90% of the total WBC except under special circumstances. 
 
SEGS and BANDS may be reported as a PERCENTAGE of the total WBC.  
Therefore the total number of neutrophils (ANC) in a give WBC is = total 
WBC  X  (% Segs  + % Bands).  This calculation will be performed by the 
Pharmacy patch to obtain and ANC when necessary
 
The sum total of SEGS% and BANDS% can never exceed 100.

not aware of this new storage arrangement while using CPRS and the MHA DLL.

perspective.

   1) YTQ GET SCRATCH: Returns data from the scratch file for the MHA DLL.

   2) YTQ KILL SCRATCH: Delete entries from the scratch file when a clinical
      reminder is saved. Input: DFN, test CODE and GIU HANDLE.
  
   3) YTQ SET SCRATCH: API is designed to set data to a scratch file for the
      MHA DLL. Input: DFN, test CODE and GIU HANDLE.
  
This patch introduces a new file called, MH CR SCRATCH FILE (#601.94). This
file is used as temporary storage space for the interface between the MHA 
DLL and Clinical Reminders.
