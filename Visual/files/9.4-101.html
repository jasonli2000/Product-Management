      DATE APPLIED:   1998-10-26 00:00:00
      APPLIED BY:   USER,ONE
      DESCRIPTION:   
      Patch makes three basic changes to the package:

  1.  Supervisor's choice of canceling a leave request is removed.
  2.  ListManager options to manage overtime (OT) warnings that appear 
      on Pay Period Exceptions report are added to the package.
  3.  Payroll Main Menu is reorganized by moving several options
      into a submenu on the Main Menu.

As item 2 above states, functionality to handle OT warnings is added.  The
patch adds a new file to store overtime warnings.  When the supervisor 
approves a time card without resolving the descrepancy between approved 
overtime for a week of the pay period and overtime calculated in the 8B 
string, then a warning will be filed in the new OVERTIME WARNINGS file.  
A new option that uses List Manager will be available to Payroll so that 
they can view these warnings.  Prior to this patch the warnings appeared 
on the Pay Period Exceptions option on the payroll main menu.  Since 
even when a corrected time card was done no change is made to the 8B 
string these warnings continue to appear on the exceptions report.  
After the patch the exceptions report will first check to see if payroll 
has cleared the warning (via the new list manager option) before 
displaying it on the exceptions report.

    30. PATCH APPLICATION HISTORY:   34 SEQ #45
      DATE APPLIED:   1999-05-10 00:00:00
      APPLIED BY:   USER,ONE
      DESCRIPTION:   
      It has been decided that the way Alternate Work Schedule Credit
Hours will be handled will be through the use of the Compensatory
Time field.  All rules governing Comp Time will be applied to Credit
Hours, e.g., pre approval and expiration after 7 pay periods, etc.  
To make the dual use of this field more clear to employees, 
supervisors and Payroll/HR, the displayed name of the field should 
be changed to reflect the actual contents of the field, Compensatory 
Time and Credit Hours.

In order to distinguish between comp time and credit hours new 
"time remarks" will be added to the TIME REMARKS file (# 457.4).  
The new remarks will be applicable to the type of time formerly 
known as comptime, but now known as comptime/credit hours.  The 
new remarks will consist of "comp time - no remarks" and "credit 
hours".  When posting comp/credit hours the timekeeper will now be 
required to enter one of the valid remarks.
