This patch will also resolve the following: 
 
1) NOIS# IOW-0997-41147 Undefined Variable DO
 
   Option: Registrant General Report [SPNL REGISTRANT GENERAL REPORT]
An undefined D0 error will occur when the user uses the option Registrant
General Report under the filtered Reports [SPNL SCD REPORTS (FILTERED)].
This problem is fixed with routine SPNLPIN. This patch is to correct an
error in the Sort by Geographical Area. The routine SPNPSR06 has been
corrected.  The routines was modified to use D0 ("0" Zero) instead of
DO ("O" Letter).
 
2) NOIS# SDC-0697-61902 Data Error in FSS report
 
   Option: Functional Status Scores [SPNF FIM SCORES]
 
When a user selects the option 2 Clinical Reported FIM, under the
selection 2 Functional Information to enter Functional Status data for SCD
patients, the user will get the wrong Extent & Completeness and Type of
Injury data displayed when the user runs the Total Score Selection option
in the Functional Status Score. Routine SPNFPLT1 has been modified to
calculate the data correctly. 
 
3) NOIS# LIT-0697-71551 Last Physical Date
         TAM-0297-30780 Last Physical Exam Field included in report.
   Option: Follow-up (Last Physical Exam)
           [SPNL F/U LAST PHYS EXAM]
 
When the user enters dates in the Sub-Fields, ANNUAL REHAB EVAL OFFERED
(#.01), ANNUAL REHAB EVAL RECEIVED (#1), and NEXT ANNUAL REHAB EVAL DUE 
(#2), under the Multiple field ANNUAL REHAB EVAL OFFERED #7 in file 154,
SCD (Spinal Cord) Registry, the option Follow-up (Last Physical Exam) under
the Filtered Reports menu should display the date in the field ANNUAL
REHAB EVAL RECEIVED #1 instead of the date in the field LAST PHYSICAL EXAM
#1.2, which is no longer used will be deleted in a future release. This is
fixed in the routines SPNLR and SPNLRE.
 
4) NOIS# DEN-0597-52404 Reg. Injury Report Wrong Sent to printer
   Option: Registrant Injury Report
           [SPNL REGISTRANT INJURY REPORT]
 
If the user enters in filter criteria to screen on SCD Patients for the
Registrant Injury Report, the program would not use the filters if that
report were queued to a printer.  This is fixed in routine SPNLPIN
 
5) NOIS# BRK-0397-12815 Extract Error
   Database Correction: Field MS Type (#2.2) in SCD (SPINAL CORD) 
                        REGISTRY File (#154)
 
The data was not deleted in the global position prior the creation of the
field MS Type #2.2 in the in SCD (SPINAL CORD) REGISTRY File (154).  The
Post- Init will check the data and delete all invalid entries.
 
NON-NOIS ISSUES
 
6) Filtered Reports menu Enhancement
 
The option Filtered Reports [SPNL SCD REPORT(FILTERED)] menu has been
modified to allow users to turn off the filters.  This enhancement will
give the users the option of using the filters for the filtered reports or
skip the filters and go directly to the reports. The following routines
have been modified: SPNPRTMT, and SPNPRTUP.
 
7) Patient Summary Report Fix
 
This patch will fix the report, Patient Summary Report [SPNL REGISTRY
SUMMARY].  The Print Template [SPNL REGISTRY DUMP] in file 154 has been
modified by replacing the old field LAST PHYSICAL EXAM with the latest
entry in field ANNUAL REHAB EVAL RECEIVED (#1) in the ANNUAL REHAB EVAL
OFFERED multiple (#7).  The new field will be listed as Last Annual Rehab
Received.

General Information
===================
 
This patch should be installed into your production UCI.  It can be
installed at anytime.  Install time should not take more than ten minutes.
This patch has no other dependencies.  No options need to be disabled.
If there are any questions, your supporting CIOFO Customer Service Patient
Information Management team will address them.
  
 
Installation instructions:                                                 
=========================                                                 
                                                                         
This Patch is created with Kernel V8 KIDS.  Follow these instructions:      
                                                                           
1) Use Packman option 'INSTALL/CHECK MESSAGE' to load the KIDS build onto    
   your system.                                                              
2) On the KIDS menu, under the 'Installation' menu, use the option 'Verify    
   Checksum in Transport Global' and verify that all routines have the      
   correct checksums.                                                        
3) From the 'Installation' Menu of KIDS, use the option 'Install             
   Package(s)'.  Select the package 'SPN*2*3' and proceed with install.      
4) If you are a MSM site, move the routines to the other servers.            
5) On a mapped system, rebuild your map set.                                  
6) Delete the post-init routine SPNPST03.
