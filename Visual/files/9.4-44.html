 Enhancements: 
 -------------
 The Consult/Request Tracking package will now support quick orders.  This
 patch will provide an option to modify existing Consult/Request Tracking
 protocols into quick orders. The new option exported is Setup
 Consult/Request Quick Order Protocols [GMRC QUICK ORDER SETUP] and this
 will be an item on the Consult/Request Management [GMRC MGR] menu.
  
 With this functionality, sites will now have the ability to pre-define
 much of the dialog encountered during ordering. 
 Sites may now: 
 1) Pre-define how service will be rendered (inpatient or outpatient)
 2) Pre-define a reason for request (may be presented for editing or
    automatically entered)
 3) Pre-define urgency of request
 4) Pre-define place of consultation
 5) Designate specific requests to bypass the ATTENTION: prompt
 6) Designate certain requests as not requiring a PROVISIONAL DIAGNOSIS
 
 These enhancements will function only if an individual procedure or
 consult protocols is placed on an add orders screen. These protocols are
 in the namespace GMRCR or GMRCT.   These protocols can be created by
 using the Setup Consult/Request Protocols [GMRC SETUP PROTOCOLS] option.
  
  
  
 The patch will also provide a new field in the REQUEST SERVICES(123.5)
 file called DAYS BEFORE PURGE (.06). This field control the number of
 days that a completed consult or procedure order for that service will
 remain in the ORDER (100) file before being purged. For those services
 not having an entry in the field, orders will be maintained for 180 days.
  
  
 The GMRC SETUP REQUEST SERVICE input template on file 123.5 has been
 modified to include the new parameter for editing when setting up a new
 service.
 
 This will now be seen when using the Setup Consult/Request Services [GMRC
 SETUP REQUEST SERVICES] option.

      DESCRIPTION:   
      CPRS internal ClearQuest ticket 17822. Remedy ticket 292869 
and 293770. Added comments during Edit/Resubmit action in CPRS GUI are 
not wrapping at 74 characters. As a result, comments are being stored on 
a single node of the Request/Processing Activity multiple in file 123.
 
Long comments are hard to read in CPRS GUI due to the need to scroll to 
the right to read the entire text.  Also for InterFacility Consults,
comments of length greater than 255 characters will not transmit via HL7.
 
This routine will search through ^GMR(123 and reformat comments when the 
following conditions are all true:
Comment is a result of the Edit/Resubmit process, NOT just Add Comment.
  Comment is longer than 80 characters AND stored on a single global node
   OR
  Comment maybe be stored on more than one global node (i.e. user entered 
  carriage returns) but has lines of text exceeding 80 characters
 
Routine will create a backup of comments before reformatting in a 
temporary array, ^XTMP("GMRC_PRE69",$J).  Temporary array is set to 
expire 45 days from the date of execution.
Routine will also send MailMan to installer containing the global 
location and text of any comments found prior to being reformatted.
