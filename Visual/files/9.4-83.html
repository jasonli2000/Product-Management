    DESCRIPTION OF ENHANCEMENTS:   
         The Controlled Substances software will monitor and track the receipt,
inventory, and dispensing of controlled substances.  Automation of the
narcotic inventory process will permit pharmacists and inspectors to
perform vault inventories utilizing portable bar code readers.  Monthly (or
more frequent) inspections can be conducted by management, with
discrepancies in stock levels automatically identified.  This software
package addresses a Congressional material weakness for Drug
Accountability.

o Batch Order Entry process
 
o Electronic Signature
 
o Nurse Dispensing with a Radio Frequency Device
 
o HL7 Interface to Narcotic Dispensing Equipment
 
o Electronic Error & Enhancement Requests (E3Rs)

      DATE APPLIED:   2000-10-04 00:00:00
      APPLIED BY:   USER,ONE
      DESCRIPTION:   
      Problem:
=======
In Outpatient Pharmacy when Rx refills are returned to stock the refill
is deleted. This deletion is not reflected in Controlled Substances 
and thus reports and Rx balances are not reflecting these changes.
 
Solution:
========
An API for Outpatient Pharmacy will be developed by Controlled Substances 
developers. This API will be called by Outpatient Pharmacy each time a 
refill is returned to stock. The API will decide whether the RX is a
controlled substance and store this information in order to inform users
of the Controlled Substances package of the status of an Rx refill.
Outpatient Pharmacy patch PSO*7*40 contains the API.
 
Problem:
=======
In some instances, two different personnel can release a CS script within
moments of each other. This cause confusion and errors in the drug's
balance.
 
Solution:
========
When a user selects a prescription, the global is instantly locked. Anyone
else trying to alter data in that prescription will be shown a message that
informs them that another user is editing that prescription.

    31. PATCH APPLICATION HISTORY:   30 SEQ #31
      DATE APPLIED:   2001-12-31 00:00:00
      APPLIED BY:   USER,ONE
      DESCRIPTION:   
      This patch addresses the problems with updating balances and releasing
controlled substances outpatient prescriptions.
  
 The functionality format is as follows:
  
1. If a controlled substances prescription fill has been released, but the
   balance NOT adjusted for that specific fill (fill not posted) in the 
   Controlled Substances (CS) V. 3.0 package, the user will be able to
   post that prescription fill to adjust the controlled substances balance
   for that drug item.
  
2. If a controlled substances prescription fill has been released and
   posted (balance adjusted), the Controlled Substances V. 3.0 package 
   will not allow any further actions to be taken on that fill. 
   In other words, a user will only be able to post (balance adjust) 
   a specific controlled substances fill only once.
  
3. When returning to stock a controlled substances prescription fill
   through the Outpatient Pharmacy V. 7.0 software, the software will
   first check to see if the user holds the PSDMGR key. If yes, the user
   will be able to return the controlled substances prescription fill
   to stock and also to update the drug balance in the Controlled 
   Substances V. 3.0 package. If no, the user will NOT be able to return 
   the controlled substances prescription fill to stock OR update the 
   drug balance in the CS package. An electronic signature will be 
   required to return a controlled substances prescription to stock and 
   to update the drug balance. 
   
   The user will now be able to post and/or release a partial fill for a 
   Consolidated Mail Outpatient Pharmacy prescription.
 
4. Users will now be able to post and/or release controlled substances
   prescriptions with a status of DISCONTINUED  from the Controlled 
   Substances V. 3.0 package.Posting and/or releasing through the 
   Controlled Substances V. 3.0 package for prescriptions with
   the statuses of DELETED, DISCONTINUED BY PROVIDER and DISCONTINUED 
   (EDIT) will not be allowed.
 
5. If a prescription fill has been returned to stock, and later processed 
   again for dispensing, the user will be able to post and/or release the 
   prescription fill through the Controlled Substances V. 3.0 package as 
   long as the new fill date is greater than the return to stock date.
 
6. The Daily Activity Log (in lieu of VA FORM 10-2320) [PSD DAILY LOG] 
   option has been enhanced to clearly identify prescriptions that have 
   been returned to stock. 
   
7. When an item is received into the Controlled Substances V. 3.0 package 
   after already being received by procurement/supply, the system does 
   not check to see if the incoming item is stocked in the pharmacy 
   location. This creates problems with the correct items because a 
   balance adjustment cannot be performed on the non-stocked item. 
   The solution to this is to check to ensure the item is stocked in the 
   selected pharmacy location.
 
8. When using the Outpatient Rx's [PSD OUTPATIENT] option, it was reported
   that the system errors out with an illegal number when the user enters
   an entry such as "501-400E097" at the "Enter/Wand PRESCRIPTION
   number" prompt. This patch corrects this problem.
 
9. It was reported that the Suremed machine is displaying the Internal
   Entry Number (IEN) of the patient instead of the Social Security Number
   (SSN). Currently, the third piece of the PID segment of the HL7 message
   is set to the IEN. This patch ensures that the patient's SSN is sent in
   the third piece of the PID segment of the HL7 message.

       When the user uses the Return Medication to Stock [PSO RETURNED STOCK]
 option to return a controlled substance to stock, the Controlled 
 Substances 3.0 package is accessed to update the drug balance. During
 this process, the problems detailed below can occur.
 
 1. After updating the balances in Controlled Substances 3.0, the 
 Outpatient Pharmacy 7.0 software requires the user to enter comments
 concerning the transaction. If the user exits the program at this point,
 the item is not marked as returned to stock in the Prescription file

 The solution for this is for an additional check on the transaction before
 allowing updating of balances in Controlled Substances 3.0.
 
 2. The second issue related to this process is that when the Controlled
 Substances 3.0 software is updating the balances, the name of the 
 dispensing site is retained. If the user then exits the Outpatient 
 Pharmacy 7.0 software, and accesses the Controlled Substances 3.0
 software, the user is not prompted for the name of the dispensing site.
 
 The solution for this is to not retain the name of the dispensing
 site.
 
 3. The third issue of this patch is on fills returned to stock, then
 re-issued. Controlled Substances 3.0 will not allow the release of 
 the new fill.
 
 The solution for this is to add additional checks on the prescription
 to ensure it is not the fill that was previously returned to stock.

      With the deployment of Cache version 5, it was discovered that the short
lock timeout values could generate a lock failure. Patch DI*22*147, which
has already been released, provided a solution to resolve this issue and
future lock issues.
 
This patch instituted a new default lock timeout of three seconds based on
a global node setting of ^DD("DILOCKTM")=3.

modifies all routines that currently use a lock timeout value of less than
3 seconds to check for the node ^DD("DILOCKTM") and to use the value
defined there, otherwise the lock timeout value is set to a default value
of 3 seconds.
