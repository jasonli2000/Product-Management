NOIS ISL-1198-52170
Test Site: Oklahoma City

This patch enables PDX to recognize and report on NKA (No Known Allergy)
when using the API EN1^GMRADPT, provided by the Adverse Reaction Tracking
package.

Adverse Reaction Tracking released a patch, GMRA*4*10, which changed the
data returned by EN1^GMRADPT if the patient has NKA.

Before the patch, the API returned:
GMRAL=0
GMRAL(<ptr to file 120.8>)=DFN_"^NKA^0^1"

After the patch, the API returned:
GMRAL=0

PDX used to grab "NKA" from the GMRAL array.  After GMRA*4*10, there was
nothing to grab.  This patch, VAQ*1.5*31, ensures that PDX recognizes that
GMRAL=0 means "NKA".
 
NOTE: This patch should be installed during off hours, when user activity
is at a minimum.  It has no patch pre-requisite requirement.

NOIS CHY-0201-51534, ASH-0501-32739, CPH-0501-40775, BHS-0601-10382,
     HIN-0701-42706, OKL-0701-70784
Test Sites: Oklahoma City, OK; Asheville, NC; Central Plains HCS

This patch prevents the sending of PDX messages to closed domains in two
ways.

1. Prevents the addition of closed domains to outgoing groups:
Screens are placed on 2 fields in the REMOTE FACILITY multiple
in file 394.83, VAQ - OUTGOING GROUP.  The fields are .01 REMOTE FACILITY
and .02 REMOTE DOMAIN.  For both of these, the DOMAIN file is checked
to ensure that the domain is not closed.  If it is, the screen fails
and you are prevented from adding the remote facility to the outgoing group.

2. Removes closed domains from outgoing groups:
When PDX messages to outgoing groups are being built, the DOMAIN file is
checked to ensure that the domain is not closed.  If it is, the remote
facility is deleted from the outgoing group, and the PDX message is not
sent to that domain.
 
NOTE: This patch should be installed during off hours, when user activity
is at a minimum.  It requires PDX patch VAQ*1.5*22.  KIDS will not check
for the existence of this patch, because it pre-dates KIDS.  The site is
responsible for performing the check.

Patch VAQ*1.5*30 exposed a problem which was not caught during its
extensive testing period.  It was not caught until it was released.
We're very sorry about that.

It is evidenced by the following:

>D ^%G

Global ^VAT(394.86
        VAT(394.86
^VAT(394.86,0) = VAQ - AUTO-NUMBERING^394.86^1^1
^VAT(394.86,1,0) = 1
^VAT(394.86,1,1) = 390
^VAT(394.86,1,2) = "1"  <----- corrupted
^VAT(394.86,1,3) = 8
^VAT(394.86,"B",1,1) = 

and by:

Global ^VAT(394.62,466
        VAT(394.62,466
^VAT(394.62,466,0) = "1"  <----- corrupted
  ...
^VAT(394.62,"B","""1""",466) =  <----- corrupted

This patch fixes that, and cleans up the corruption in file 394.86.
